# Strategic Planning Module - Design Approach

**Version:** 1.0  
**Date:** December 22, 2025  
**Status:** Approved Concept  

---

## Executive Summary

This document captures the approved design approach for reimagining the Strategic Planning module. The core insight is that **Planning** and **Tracking** are fundamentally different mental modes that should be separated into distinct experiences.

---

## The Core Insight

| Aspect | Planning Mode | Tracking Mode |
|--------|---------------|---------------|
| **Frequency** | Quarterly/Annual | Daily/Weekly |
| **Mindset** | Strategic, deliberate | Tactical, rapid |
| **Question** | "What should we do?" | "How are we doing?" |
| **Actions** | Define, align, decide | Update, adjust, execute |
| **Time** | Future-focused | Present-focused |

Mixing them creates cognitive overhead and a cluttered experience.

### Step-by-Step Flow

| Step | Name | Purpose | Key Features |
|------|------|---------|--------------|
| 1 | **Intent Canvas** | Define WHAT | Goal statement, why it matters, AI alignment check |
| 2 | **Strategy Workshop** | Define HOW | Add strategies, AI suggestions, alignment scores |
| 3 | **Success Measures** | Define KPIs | Per-strategy KPIs + goal-level KPIs, target setting |
| 4 | **Action Planning** | Define TASKS | Per-strategy actions, AI suggestions, assignments |
| 5 | **Review & Activate** | Launch | Summary view, share with team, "Activate Goal" |

### Key UX Elements

- **Progress stepper** - Always know where you are
- **Save Draft** - Never lose work
- **AI assistance** - At every step, not intrusive
- **Drag-to-reorder** - Prioritize strategies
- **Expand/collapse** - Progressive detail

## Key Design Principles

### 1. Clear Mode Separation
- Strategic Planning = **Building the plan** (less frequent, deliberate)
- Operations = **Executing the plan** (frequent, rapid)
- Visual distinction: maybe Strategic Planning has a blue accent, Operations has green

### 2. Wizard for Planning, Dashboard for Tracking
- Planning Studio guides you step-by-step
- Command Center shows what matters NOW

### 3. Progressive Disclosure
- Summary first, details on demand
- Expand/collapse everywhere
- Slide-out panels for context

### 4. Action-Oriented Design
- Every screen answers: "What should I do next?"
- Quick entry buttons everywhere
- Minimize clicks for common tasks

### 5. AI as Assistant, Not Replacement
- AI suggestions are optional, not required
- Always show reasoning ("Why this suggestion?")
- User is always in control

## UI Preferences

Based on stakeholder feedback:

1. **Slide-out panels from the right** preferred over floating modals
2. **Expanding cards** for progressive detail
3. **Modern, clean aesthetic** with ample whitespace
4. **Color-coded status indicators** for quick scanning
5. **Inline editing** where possible to reduce context switches
